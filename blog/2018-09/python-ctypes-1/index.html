在Python中使用C/C++——ctypes(1)

发表于 2018-09-02 | 分类于 Python | 评论数： | 阅读次数：

ctypes是什么

ctypes is a foreign function library for Python. It provides C compatible data types, and allows calling functions in DLLs or shared libraries. It can be used to wrap these libraries in pure Python.

简单来说，ctypes是一个让开发者在Python中调用C/C++动态链接库的外部函数库，和JNI一样，一般有三种场景需要我们在Python中使用C/C++编写的库：

  1. 有大量现存的库或SDK是用C/C++编写的，如果想要在Python中使用其功能，就不得不使用ctypes这样的库来进行调用
  2. 需要与Python标准库还没有Wrap的操作系统API交互或者进行一些底层操作，比如内存分配（当然也可以直接使用C编写扩展模块，但是ctypes的使用更加简单）
  3. Python的执行效率很低，在进行计算密集型的任务时，可以使用C/C++来提高执行效率（使用PyPy、numba这样的JIT编译器来提高运行效率也是一种选择）

准备C/C++动态链接库

既然要在Python中使用动态链接库，首先就来看一下如何编写并编译一个，使用的环境为macOS 10.13，编译器版本为Apple LLVM version 9.1.0。

extern "C"

如果是使用C++的话，需要将头文件的函数声明包含在extern "C"中，这个声明在编写C++库的时候非常常用，因为C++编写的库很有可能被C程序使用，而在C++中有着函数重载这一特性，所以编译后的文件中函数名将会加上参数类型列表，这个不同将导致C语言和C++无法互相调用函数。

extern "C"包含extern和"C"两部分。extern是C/C++中的关键字，表示被标记的变量或方法可以在外部模块被使用，"C"告诉编译器使用C-linkage进行编译声明了extern "C"的代码。一般会使用以下的头文件模板：

__cdecl和__stdcall

__cdecl是C/C++默认的调用约定，__stdcall调用约定一般用于调用Win32 API，这两个函数的参数压栈顺序是相同的（从右往左）。两者最重要的不同点在于清理堆栈，__cdecl为调用者负责清理堆栈，__stdcall为被调用者清理堆栈。之所以提到这两种不同的调用约定，是因为在ctypes中对于这两者需要CDLL和WinDLL两种不同的调用方法来加载库。

如何编译库

大部分IDE都可以新建一个库项目，在编写完成代码后编译即可得到一个库文件。也可以直接使用编译器编译生成库文件，对于*nix操作系统的用户，可以使用gcc或者clang等编译器（Windows用户则为MinGW或VC等编译器），使用以下的命令即可编译生成一个库文件。

-fPIC表示编译将生成位置无关代码，这表示使用编译出的代码访问常量时将使用GOT（global offset table,全局偏移表）得到地址。-shared表示编译生成的对象可以被其他可执行程序共享，一般使用时需要同时加上-fPIC。

c_char_p、c_wchar_p、c_void_p修改值的时候将会修改指向的地址，而指向的内存块是不可变的，这和Python字符串不可变相同。如果需要使用可变内存块，应该使用create_string_buffer()或create_unicode_buffer()（用于wchart）。

函数调用

ctypes可以自动将整数（None, integers, longs）、字符串、unicode字符串转换为C数据类型，比如None转换为NULL，字符串转换为字符指针指向的内存块。但是其他类型如浮点数，就需要显式的包裹在ctypes类型中来将它们转换为C数据类型。

在函数调用中使用自定义的类型时，ctypes将会查找_as_parameter_属性作为函数参数，同样的，这个属性的类型应该是整数、字符串、unicode字符串。

参数类型和返回类型

可以给函数增加一个argtypes属性来指定函数的参数类型，当调用函数时传入的参数与指定参数类型不一致时，会抛出ArgumentError。如果要将自定义类放在argtypes里需要在自定义类中实现from_param()方法。

ctypes调用函数时，默认的返回值是C的int类型，如果是其他返回类型要用restype属性手动指定，比如divide()方法的返回值为double类型，就要给restype赋值一个c_double。如果返回类型是整数，还可以给restype赋值一个callable对象，函数返回值将会作为参数传给callable对象，这种用法在检测函数调用出错时很有用。
